# 移动应用开发题库刷题网页设计

## 背景

当前目录包含 6 组题库截图，共 140 题：

- `测验1`：40 张截图，已有 `[OCR]_测验1_20260704_1320.txt`
- `测验2`：20 张截图
- `测验3`：20 张截图
- `测试4`：20 张截图
- `测试5`：20 张截图
- `测试6`：20 张截图

`测验1` 的现有 OCR 文本可以作为底稿，但不能直接用于网页题库。它能覆盖 40 题的大部分结构信息，但存在噪声和错字，例如 `pages.ison`、`is文件`、`Al讲解`，第 14 题的正确答案区域被底部按钮遮挡。最终题库需要经过结构化整理和人工可推断修正。

## 目标

制作一个本地静态刷题网页，用于练习移动应用开发题库。网页应支持：

- 覆盖全部 140 题
- 按原文件夹分套刷题
- 合并全部题库随机刷题
- 练习模式和考试模式切换
- 只保存错题本，不保存完整答题记录
- 不显示原截图，只显示整理后的题干、选项、答案和解析

## 非目标

- 不做用户登录、后端服务或数据库
- 不展示原始截图
- 不保存完整练习历史
- 不追求题库在线同步
- 不把 OCR 临时预览文件作为交付内容

## 数据设计

题库整理成 `data/questions.json`，每题一条结构化记录：

```json
{
  "id": "quiz1-001",
  "set": "测验1",
  "number": 1,
  "type": "single",
  "question": "在uni-app中，配置tabBar的文件是？",
  "options": [
    { "key": "A", "text": "app.vue" },
    { "key": "B", "text": "main.js" },
    { "key": "C", "text": "pages.json" },
    { "key": "D", "text": "manifest.json" }
  ],
  "answer": ["C"],
  "explanation": "tabBar的配置需要在pages.json文件的tabBar节点中进行设置。",
  "source": "测验1"
}
```

题型统一为：

- `single`：单选题，答案数组长度为 1
- `multiple`：多选题，答案数组长度大于 1
- `judge`：判断题，选项统一为 `A. 对`、`B. 错`

整理策略：

- `测验1` 使用现有 OCR 文本作为底稿，并修正明显 OCR 错字
- 其他 5 组从截图中提取并整理为同样结构
- 对被遮挡但可通过得分和我的答案可靠推断的题，写入推断后的正确答案
- 对无法可靠推断的题，在整理阶段核对截图后再进入题库
- 网页运行时不显示“疑似 OCR 不确定”状态

## 界面设计

采用混合版布局：

- 顶部：应用标题、练习/考试模式切换、错题本入口
- 左侧：题库范围选择，包括全部随机、各套题、错题本
- 中央：当前题目、选项、反馈、上一题/下一题
- 右侧：练习模式显示统计信息；考试模式显示答题卡、已答数量和交卷按钮

响应式规则：

- 桌面端使用三栏布局
- 窄屏设备折叠为单栏，题库选择和答题卡变为顶部或抽屉式区域
- 文本不能与按钮、选项、解析区域重叠

## 交互设计

### 范围选择

- `全部随机`：从 140 题中打乱顺序
- 单套题：按原题序显示
- `错题本`：只显示当前浏览器本地保存的错题

### 练习模式

- 用户选择答案后立即判题
- 显示正确/错误、正确答案和解析
- 答错自动加入错题本
- 错题本中的题答对后不自动删除，提供“移出错题”按钮
- 支持上一题和下一题

### 考试模式

- 作答过程中不显示解析和正确答案
- 支持上一题、下一题和答题卡跳转
- 交卷后显示得分、错题列表和每题解析
- 答错题自动加入错题本

### 错题本

- 使用浏览器 `localStorage` 保存错题 ID
- 只保存错题集合，不保存完整答题记录
- 支持移出单题
- 支持清空错题本
- 支持按题库范围查看错题

## 技术设计

交付为静态网页项目：

- `index.html`：页面结构
- `styles.css`：界面样式和响应式布局
- `app.js`：题库加载、模式切换、判题、交卷、错题本逻辑
- `data/questions.json`：140 题结构化数据
- `tools/`：可选，一次性整理脚本或辅助数据，不作为网页运行依赖

运行方式：

- 可以直接通过本地静态服务器打开
- 不需要后端、不需要登录、不需要数据库

## 验收标准

- `data/questions.json` 包含 140 题
- 每题都有 `id`、`set`、`number`、`type`、`question`、`options`、`answer`、`explanation`
- 单选、多选、判断题判题正确
- 全部随机会打乱题序
- 单套题按原题序显示
- 练习模式选完立即反馈
- 考试模式交卷后显示分数和解析
- 答错题会加入错题本
- 错题本刷新页面后仍保留
- 可以移出单题和清空错题本
- 桌面和手机宽度下布局可用且文字不重叠

## 风险与处理

- OCR 错字风险：结构化整理时对关键字段进行核对，修正常见错字，例如 `pages.ison -> pages.json`、`is文件 -> js文件`
- 答案遮挡风险：若可由得分和我的答案可靠推断，则写入正确答案；不可推断时核对截图
- 数据录入风险：实现题库校验脚本，检查题号、题型、选项数量和答案合法性
- 本地存储风险：错题本只保存在当前浏览器，换浏览器或清理缓存会丢失；这是当前设计接受的限制

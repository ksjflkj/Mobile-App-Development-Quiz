# 移动应用开发题库网页刷题系统重构设计规范

## 背景
用户需要重构现有的移动应用开发题库网页刷题系统。
该系统目前是一个纯静态页面，功能包含：
- 从 `data/questions.json` 加载 140 道题目。
- 区分“练习模式”和“考试模式”。
- 支持单元选择、随机抽题、错题本（基于 `localStorage` 缓存）。

## 核心重构目标
为了满足用户的最新诉求，重构将聚焦于以下两点：
1. **点开即用（零配置直接运行）**：
   通过消除 `fetch()` 本地 JSON 带来的 CORS 跨域报错，使用户双击 `index.html` 即可在本地浏览器（使用 `file://` 协议）中流畅答题。
2. **现代仪表盘化（Dashboard）与新模式引入**：
   - 首页展示题库各单元进度卡片和错题统计。
   - 增加三种答题模式：单元测试、随机抽题和**限时测验（带倒计时，超时自动交卷）**。
   - 升级为极其美观、带有呼吸感的现代 UI，提供深浅色主题自适应、毛玻璃效果和精致微动画。

## 架构与文件结构设计

我们将重构后的项目结构设计如下：

```
移动应用开发题库/
├── index.html                  # 单页应用（SPA）入口骨架，包含首页和答题区
├── styles.css                  # 包含现代 CSS 变量、布局样式及所有微动画、主题系统
├── data/
│   ├── questions.json          # 原始题库备份（仅作数据保留）
│   └── questions-db.js         # 新的数据源，将题库作为全局变量挂载至 window.QUIZ_QUESTIONS
└── src/
    └── quiz-core.js            # 核心业务逻辑（挂载在 window.QuizCore 下）
```

### 规避跨域设计（零环境依赖）
为了保证 100% 的双击即用且不触发浏览器的 CORS 跨域报错，我们**不采用** ES Modules (`type="module"`)，而是使用传统的脚本加载方式：
1. 在 `index.html` 中通过普通的 `<script>` 标签依次加载：
   - `<script src="./data/questions-db.js"></script>`
   - `<script src="./src/quiz-core.js"></script>`
   - `<script src="./app.js"></script>`
2. 彻底删除 JS 代码中的 `import` 和 `export` 关键字。
3. `quiz-core.js` 中的核心计算方法将全部挂载在全局变量 `window.QuizCore`（或直接暴露在全局作用域）中，由 `app.js` 调用。
4. 这样双击 `index.html` 即可立即加载所有资源，规避一切本地跨域报错。

## UI/UX 视觉与布局设计

系统采用单页视图切换机制（通过控制 `.view-container` 容器的显隐实现无缝切换）。

### 1. 首页控制台 (Home Dashboard)
- **头部 Header**：标题“移动应用开发题库”；右侧为“主题切换”按钮。
- **统计看盘 (Stats Row)**：
  - **题库总览**：展示题库总题数与用户累计答题数。
  - **错题本卡片**：醒目的橙色渐变发光卡片，显示错题本内题目数，点击一键进入错题复习。
- **练习/测验快捷面板**：
  - **随机抽题按钮**：点击弹出选择框（10题 / 20题 / 40题），确认后进入随机抽题。
  - **限时测验按钮**：点击弹出测验配置弹窗，可选择题量及限时（如 15分钟 / 30分钟 / 45分钟），确认后开启考试。
- **单元列表网格 (Unit Test Grid)**：
  - 每一个单元（如“测验1”、“测试4”）渲染为一个独立的精致卡片。
  - 卡片包含：单元标题、题目数、**圆角进度条**（利用 `localStorage` 记录已答题目，计算已答百分比）以及“进入单元”的悬浮按钮。

### 2. 答题工作区 (Quiz Workspace)
- **双栏响应式布局**：
  - **左侧/顶部**：答题控制栏。包含“返回首页”按钮、当前答题进度（已答/总数）、倒计时显示器（仅在限时测验下显示）、答题卡数字矩阵。
  - **右侧/中央**：题目卡片。
- **题目卡片细节**：
  - **元信息**：以小字徽标（Badge）形式展示“单选题/多选题/判断题”及所属单元。
  - **题干**：20px 粗体展示。
  - **选项列表**：
    - 每个选项为独立卡片。悬停时有微移（translate-x）与阴影加深效果。
    - 选中态：淡蓝色边框与背景。
    - 练习模式判题态：正确选项显示为淡绿色，错误选择显示为淡红色。
  - **解析面板 (Feedback Panel)**：在练习模式下，选中后滑入展出，显示“答对/答错”的醒目标签、正确答案和详细解析。

## 核心业务交互设计

### 1. 刷题/测验模式划分
- **单元测试 (Unit Test)**：
  - 默认采用“练习模式”，支持即时反馈。
  - 题目按原题序排列。
  - 答错自动加入错题本，提供手动移出错题功能。
- **随机抽题 (Random Quiz)**：
  - 打乱题序，抽取指定题量。
  - 采用“练习模式”。
- **限时测验 (Timed Exam)**：
  - 打乱题序，抽取指定题量。
  - 开启倒计时。
  - 作答过程中**不显示**即时反馈与解析。答题卡只标记“已作答/未作答”。
  - 支持中途手动交卷或超时强制交卷。交卷后统一结算，进入“结算报告”。

### 2. 倒计时与自动交卷
- 使用 `setInterval` 在前端驱动时间递减。
- 时钟低于 3 分钟时展示醒目的红色高亮和闪烁特效。
- 时间归零时，立即触发交卷函数，计算得分，将做错的题目自动加入错题本，并弹出“考试结算报告”窗口。

### 3. 本地存储 (Local Storage) 设计
- `mobile-quiz-wrong-ids`: 存储错题 ID 数组。
- `mobile-quiz-progress`: 存储已作答题目的 ID 映射表，结构为 `{ "setName": [ "id1", "id2" ] }`，用于计算单元进度条。
- `mobile-quiz-theme`: 存储用户的主题设置（`light` 或 `dark`）。

## 验收与测试标准

1. **直接双击运行验证**：直接双击 `index.html`，无需开启任何本地 HTTP 服务，页面应能正常加载题库，不能有任何 CORS 报错。
2. **首页进度条验证**：在“测验1”中回答 5 道题后返回首页，“测验1”卡片的进度条应更新。
3. **倒计时超时验证**：开启 1 分钟测验，等待时间归零，验证是否会自动交卷并弹出结算卡。
4. **错题本联动验证**：在练习模式下答错一题，验证错题本数量是否加 1；进入错题本做对后手动移出，验证错题本数量是否减少。
5. **多端自适应验证**：在手机宽度（<768px）和电脑宽屏下，各区域应自适应排列，文字无遮挡和重叠。
